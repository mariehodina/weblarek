# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`

#### Card
базовый класс для всех типов карточек товара (в каталоге, в корзине, в модальном окне). наследуется от Component и выносит общий функционал для всех трёх видов карточек.

конструктор
`constructor(container: HTMLElement, events: EventEmitter)` - принимает корневой элемент карточки и экземпляр брокера событий.

поля класса
`titleElement: HTMLElement` - элемент для отображения названия товара.
`priceElement: HTMLElement` - элемент для отображения цены товара.
`events: EventEmitter` - брокер событий для генерации уведомлений.

методы
`render(data: T): HTMLElement` - заполняет карточку общими данными (название, цена). возвращает корневой элемент.
`setTitle(value: string): void` - устанавливает название товара.
`setPrice(value: number | null): void` - устанавливает цену (null отображается как «бесценно»).


#### Form
базовый класс для всех форм (оформление заказа, контактные данные). так же наследуется от Component и выносит общий функционал для всех форм.

конструктор
`constructor(container: HTMLElement, events: EventEmitter)` - принимает корневой элемент формы и экземпляр брокера событий.

поля класса
`formElement: HTMLFormElement` - элемент формы.
`submitButton: HTMLButtonElement` - кнопка отправки формы.
events: EventEmitter - брокер событий.
`errorElement: HTMLElement` - элемент для отображения ошибок валидации.

методы
`render(data?: Partial<T>): HTMLElement` - обновляет отображение формы.
`setValid(isValid: boolean): void` - включает/выключает кнопку отправки в зависимости от валидности.
`showError(error: string): void` - отображает сообщение об ошибке.
`clearError(): void` - очищает сообщение об ошибке.


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события   подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

## Данные
 модели данных инкапсулируют бизнес-логику, связанную с товарами, корзиной и информацией о покупателе, и не зависят от способа отображения этих данных. все изменения в моделях сопровождаются генерацией событий, которые уведомляют другие слои приложения (презентер и представления) о необходимости обновления.

### модели данных
классы-модели реализуют хранение данных и операции над ними. каждая модель имеет четко определенную зону ответственности и минимально необходимый набор методов для управления своими данными. при изменении состояния модели генерируют соответствующие события, которые передаются через брокер событий.

#### Catalog
модель каталога отвечает за хранение и управление списком товаров, доступных в магазине, за отслеживание товара, выбранного пользователем для детального просмотра. класс обеспечивает доступ к данным о товарах и их выборку по идентификатору.

конструктор:
`constructor(events: EventEmitter)` - принимает экземпляр брокера событий для генерации уведомлений об изменениях в модели.

поля класса: 
`_products: IProduct[]` - массив всех товаров, загруженных с сервера. хранит полный каталог доступных товаров.
`_selectedProduct: IProduct | null `- товар, который в текущий момент выбран для просмотра в окне. изначатльно значение - null.

методы:
`setProducts(products: IProduct[]): void `- сохраняет массив товаров в модел, принимает массив объектов товаров, полученных от сервера. после сохранения генерирует событие catalog:changed для обновления визуала.
`getProducts(): IProduct[]` - возвращает массив всех товаров из каталога. в данном случае используется для отображения списка товаров.
`getProductById(id: string): IProduct | undefined` - находит, а, после, возвращает товар по его же идентификатору. возвращает undefined в случае, если товар не обнаружен.
`setSelectedProduct(product: IProduct): void` - сохраняет переданный товар как выбранный для дальнейшего просмотра; генерирует событие product:selected для отображения полной информации.
`getSelectedProduct(): IProduct | null `- возвращает выбранный для просмотра товар. 


#### ShoppingСart
модель корзины управляет состоянием корзины покупателя: хранит добавленные товары, не допускает дублирование, рассчитывает общую стоимость и количество товаров. этот класс обеспечивает все операции, связанные с созданием заказа пользователем.

конструктор:
`constructor(events: EventEmitter)` - принимает экземпляр брокера событий для генерации уведомлений о каких-либо изменениях в корзине.

поля класса: 
`_items: IProduct[]` - массив товаров, что пользователь добавил в корзину. каждый товар хранится в едином экземпляре.

методы:
`getItems(): IProduct[] `- возвращает массив товаров, находящихся в корзине; используется для отображения списка выбранных пользователем товаров.
`addItem(product: IProduct): void` - добавляет товар в корзину в случае, если этого товара там еще нет; проверяет наличие товара по идентификатору перед добавлением. Генерирует событие cart:changed после добавления.
`removeItem(productId: string): void` - Удаляет товар из корзины по его идентификатору. Генерирует событие cart:changed после удаления.
`clear(): void` - Полностью очищает корзину, удаляя все товары. Генерирует событие cart:changed для обновления интерфейса.
`getTotalPrice(): number` - Вычисляет и возвращает общую стоимость всех товаров в корзине. Товары с ценой null (бесценные) исключаются из подсчета.
`getItemCount(): number` - Возвращает количество товаров в корзине. Используется для отображения счетчика в иконке корзины.
`containsItem(productId: string): boolean` - Проверяет, есть ли товар с указанным идентификатором в корзине. Используется для управления состоянием кнопки "Добавить в корзину".


#### Buyer
модель покупателя хранит данные, введенные пользователем в форме оформления заказа, и управляет их валидацией. этот класс обеспечивает пошаговое заполнение формы и проверку корректности введенных данных перед отправкой заказа.


конструктор:
`constructor(events: EventEmitter)` - принимает экземпляр брокера событий для генерации уведомлений об изменениях в данных пользователя.

поля класса:
`_data: IBuyer` - объект, хранящий текущие данные покупателя. Начальное значение: { payment: '', email: '', phone: '', address: '' }. Каждое поле обновляется отдельно для поддержки пошагового заполнения формы.

методы:
`setField<T extends keyof IBuyer>(field: T, value: IBuyer[T]): void` - Устанавливает значение для одного из полей покупателя (например, email). Позволяет обновлять поля по отдельности без потери данных других полей. Генерирует событие buyer:changed после обновления.
`setData(data: Partial<IBuyer>): void` - Обновляет одно или несколько полей данных покупателя. Принимает частичный объект с данными, что позволяет обновлять поля группами.
`getData(): IBuyer` - Возвращает полный объект с данными покупателя. Используется при отправке заказа на сервер.
`clear(): void` - Сбрасывает все данные покупателя к начальному состоянию. Генерирует событие buyer:changed после очистки.
`validate(): Partial<Record<keyof IBuyer, string>>` - Проверяет валидность каждого поля. Возвращает объект, где ключами являются названия полей, а значениями — текст ошибки. Если поле валидно, оно отсутствует в возвращаемом объекте. Используется представлением формы для отображения ошибок валидации.


### слой коммуникации
конструктор:

#### ContactsFormData
Интерфейс данных для формы контактных данных.

#### ContactsForm
Компонент формы контактных данных. Наследуется от Form. Отвечает за ввод email и телефона.

конструктор
`constructor(container: HTMLElement, events: EventEmitter)` - принимает корневой элемент формы и брокер событий.

поля класса
`emailInput: HTMLInputElement` - поле ввода email.
`phoneInput: HTMLInputElement` - поле ввода телефона.

методы
`render(data: ContactsFormData): HTMLElement` - обновляет отображение формы, генерирует событие contacts:change при изменении любого поля, а так же возвращает корневой элемент.


#### Modal
Класс модального окна, Не имеющий дочерних классов. он управляет отображением модального окна и его содержимог, где вся разметка, отображаемая внутри, — это самостоятельные компоненты, которые передаются извне.

конструктор
`constructor(container: HTMLElement, events: EventEmitter)` - принимает корневой элемент модального окна и брокер событий.

поля класса
`modalElement: HTMLElement` - корневой элемент модального окна.
`contentElement: HTMLElement` - контейнер для динамического содержимого.
`closeButton: HTMLButtonElement` - кнопка закрытия.
`events: EventEmitter` - брокер событий.

методы
`open(): void `- открывает модальное окно. 
`close(): void` - закрывает модальное окно. 
`setContent(content: HTMLElement): void `- устанавливает содержимое модального окна.
`render(data?: object): HTMLElement` - возвращает корневой элемент модального окна.

# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`

#### Card
базовый класс для всех типов карточек товара (в каталоге, в корзине, в модальном окне). наследуется от Component и выносит общий функционал для всех трёх видов карточек.

конструктор
`constructor(container: HTMLElement)` - принимает корневой элемент карточки 

поля класса
`titleElement: HTMLElement` - элемент для отображения названия товара.
`priceElement: HTMLElement` - элемент для отображения цены товара.

методы
`setTitle(value: string): void` - устанавливает название товара.
`setPrice(value: number | null): void` - устанавливает цену (null отображается как «бесценно»).


#### Form
базовый класс для всех форм (оформление заказа, контактные данные). 

конструктор
`constructor(container: HTMLElement, events: EventEmitter)` - принимает корневой элемент формы и экземпляр брокера событий.

поля класса
`submitButton: HTMLButtonElement` - кнопка отправки формы.
events: EventEmitter - брокер событий.
`errorElement: HTMLElement` - элемент для отображения ошибок валидации.

методы
`setValid(isValid: boolean): void` - включает/выключает кнопку отправки в зависимости от валидности.
`showError(error: string): void` - отображает сообщение об ошибке.


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события   подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

## Данные
 модели данных инкапсулируют бизнес-логику, связанную с товарами, корзиной и информацией о покупателе, и не зависят от способа отображения этих данных. все изменения в моделях сопровождаются генерацией событий, которые уведомляют другие слои приложения (презентер и представления) о необходимости обновления.

### модели данных
классы-модели реализуют хранение данных и операции над ними. каждая модель имеет четко определенную зону ответственности и минимально необходимый набор методов для управления своими данными. при изменении состояния модели генерируют соответствующие события, которые передаются через брокер событий.

#### Catalog
модель каталога отвечает за хранение и управление списком товаров, доступных в магазине, за отслеживание товара, выбранного пользователем для детального просмотра. класс обеспечивает доступ к данным о товарах и их выборку по идентификатору.

конструктор:
`constructor(events: IEvents)` - принимает брокер событий

поля класса: 
`_products: IProduct[]` - массив всех товаров, загруженных с сервера. хранит полный каталог доступных товаров.
`_selectedProduct: IProduct | null `- товар, который в текущий момент выбран для просмотра в окне. изначатльно значение - null.

методы:
`setProducts(products: IProduct[]): void` — сохраняет массив товаров, генерирует `item:changed`
`getProducts(): IProduct[]` — возвращает массив товаров
`getProduct(id: string): IProduct | undefined` — возвращает товар по ID
`setSelectedProduct(product: IProduct): void` — сохраняет выбранный товар
`getSelectedProduct(): IProduct | null` — возвращает выбранный товар


#### Basket

Модель корзины.

Конструктор
`constructor(events: IEvents)` — принимает брокер событий

Поля класса
`items: IProduct[]` — массив товаров в корзине

Методы
`getItems(): IProduct[]` — возвращает массив товаров
`addItem(item: IProduct): void` — добавляет товар, генерирует `basket:changed`
`removeItem(id: string): void` — удаляет товар, генерирует `basket:changed`
`clear(): void` — очищает корзину, генерирует `basket:changed`
`getTotal(): number` — возвращает общую сумму
`getCount(): number` — возвращает количество товаров
`contains(id: string): boolean` — проверяет наличие товара


#### Buyer
модель покупателя хранит данные, введенные пользователем в форме оформления заказа, и управляет их валидацией. этот класс обеспечивает пошаговое заполнение формы и проверку корректности введенных данных перед отправкой заказа.


конструктор:
`constructor(events: EventEmitter)` - принимает экземпляр брокера событий для генерации уведомлений об изменениях в данных пользователя.

поля класса:
`_data: IBuyer` - объект, хранящий текущие данные покупателя. Начальное значение: { payment: '', email: '', phone: '', address: '' }. Каждое поле обновляется отдельно для поддержки пошагового заполнения формы.

методы:
`setField<T extends keyof IBuyer>(field: T, value: IBuyer[T]): void` - Устанавливает значение для одного из полей покупателя (например, email). Позволяет обновлять поля по отдельности без потери данных других полей. Генерирует событие buyer:changed после обновления.
`setData(data: Partial<IBuyer>): void` - Обновляет одно или несколько полей данных покупателя. Принимает частичный объект с данными, что позволяет обновлять поля группами.
`getData(): IBuyer` - Возвращает полный объект с данными покупателя. Используется при отправке заказа на сервер.
`clear(): void` - Сбрасывает все данные покупателя к начальному состоянию. Генерирует событие buyer:changed после очистки.
`validate(): Partial<Record<keyof IBuyer, string>>` - Проверяет валидность каждого поля. Возвращает объект, где ключами являются названия полей, а значениями — текст ошибки. Если поле валидно, оно отсутствует в возвращаемом объекте. Используется представлением формы для отображения ошибок валидации.


### слой коммуникации
конструктор: 
`constructor(api: IApi)` - принимает экземпляр класса Api, что реализует IApi, для выполнения запросов к серверу.

Методы:
`getProducts(): Promise<IProduct[]>` - возвращает массив товаров, полученных от сервера. при успешном ответе сервер возвращает объект с полем items, содержащим массив товаров.
`sendOrder(order: IOrder): Promise<IOrderResult>` - принимает объект заказа, содержащий данные покупателя и выбранные товары. Возвращает объект с подтверждением покупки, содержащий id заказа и total сумму.

`constructor(container: HTMLElement, actions?: ICardCatalogActions)` - принимает родительский элемент карточки и экземпляр брокера событий для генерации уведомлений о нажатии.

методы:
